# 🌳 성령의 나무 — 시스템 안내서

> 개발을 처음 접하는 분도 이해할 수 있도록 아주 쉽게 설명한 문서입니다.
> 어려운 용어가 나오면 바로 옆에 쉬운 비유를 같이 적어뒀어요.

---

## 1. 한 줄 요약

**선생님이 학생에게 "달란트(칭찬 점수)"를 주고, 학생이 그 달란트를 모아 "공동체 나무"에 기부하면, 모두의 기부가 모여 큰 나무가 무럭무럭 자라는** 웹사이트예요.

비유하자면 — **반 전체가 함께 키우는 칭찬 스티커 화분** 🌱 입니다.
- 선생님 = 칭찬 스티커(달란트)를 주는 사람
- 학생 = 스티커를 모아서 화분에 보태는 사람
- 공동체 나무 = 모두의 스티커가 합쳐져 자라는 화분

---

## 2. 누가 쓰나요? (3가지 역할)

| 역할 | 하는 일 | 비유 |
|---|---|---|
| **학생** | 받은 달란트를 확인하고, 공동체 나무에 기부 | 용돈 받아서 저금통에 넣는 사람 |
| **선생님** | 학생에게 달란트를 줌 (칭찬) | 용돈 주는 어른 |
| **관리자** | 계정 만들기/관리, 전체 현황 관리 | 은행 지점장 |

---

## 3. 어떻게 동작하나요? (전체 흐름)

```
①선생님이 학생에게 달란트를 줌
        ↓
②학생에게 달란트가 쌓임 (예: 5개 받음)
        ↓
③학생이 "기부하기" 버튼으로 공동체 나무에 기부 (예: 3개)
        ↓
④모든 학생의 기부가 합쳐짐
        ↓
⑤합계가 일정 개수를 넘으면 나무가 한 단계 성장! 🌱→🌿→🌳→🎆
```

### 핵심 규칙 3가지

**📏 규칙 1 — 나무가 자라는 기준 (4단계)**
모두의 기부 합계가 아래만큼 모이면 나무가 자랍니다.

| 누적 기부 | 나무 단계 |
|---|---|
| 0~1개 | 🌱 1단계 (씨앗) |
| **2개** | 🌿 2단계 |
| **5개** | 🌳 3단계 |
| **8개** | 🎆 4단계 (활짝!) |

**🚦 규칙 2 — 하루 한도**
한 학생은 **하루에 최대 15개**까지만 달란트를 받을 수 있어요. (너무 몰아주는 걸 막기 위함)

**🕵️ 규칙 3 — 기부자는 익명**
누가 기부했는지 아무도 모르게, 기부하면 "멋쟁이 사자", "용감한 토끼" 같은 **랜덤 별명**으로만 표시돼요. (실명은 절대 안 보임)

## 6. 시스템은 무엇으로 만들어졌나요? (초보자용 구조 설명)

웹사이트 하나는 보통 **3개의 부품**으로 나뉘어요. 식당에 비유하면 이해가 쉬워요. 🍽️

```
[ 손님(사용자) ]
       │  주문/음식
       ▼
┌─────────────────┐   ┌──────────────────┐   ┌─────────────────┐
│  ① 프론트엔드    │   │  ② 백엔드         │   │  ③ 데이터베이스  │
│  (화면/메뉴판)   │◄─►│  (주방/요리사)    │◄─►│  (창고/냉장고)   │
│  React + Vite    │   │  Django(파이썬)   │   │  PostgreSQL     │
│  ☁ Vercel에 배포 │   │  ☁ Render에 배포  │   │  ☁ Neon에 보관  │
└─────────────────┘   └──────────────────┘   └─────────────────┘
```

| 부품 | 하는 일 | 비유 | 어디에 올렸나(배포) |
|---|---|---|---|
| **① 프론트엔드** | 사람이 보는 화면. 버튼·나무·숫자 표시 | 메뉴판 + 홀 | **Vercel** (무료) |
| **② 백엔드** | 규칙 처리. "달란트 줘", "기부해" 같은 요청을 받아 계산 | 주방 요리사 | **Render** (무료) |
| **③ 데이터베이스** | 모든 기록 저장 (누가 몇 개 받고 기부했는지) | 창고/냉장고 | **Neon** (무료) |

**손님(학생)이 "기부하기"를 누르면:**
1. ① 화면이 ② 주방에 "이 학생이 3개 기부한대요" 라고 전달
2. ② 주방이 규칙 확인(잔액 충분한지 등) 후 ③ 창고에 기록 저장
3. ② 주방이 ① 화면에 "완료!" 응답 → 화면의 숫자·나무가 갱신

> **왜 3개로 나누나요?** 화면(디자인), 규칙(로직), 기록(데이터)을 따로 관리하면 한 곳을 고쳐도 다른 곳이 안 망가져서 안전하고 편해요.

## 7. 자주 하는 작업 (관리자용)

### ✅ 진짜 학생/선생님 계정 추가하기
지금은 데모 계정(2+2)만 있어요. 실제 인원을 추가하려면:
1. https://holytree-api.onrender.com/admin/ 접속 → `관리자 / admin1234` 로그인
2. 왼쪽 **Users(사용자)** 클릭 → 오른쪽 위 **ADD USER**
3. 아이디(이름)·비밀번호 입력 → 저장
4. 다음 화면에서 **역할(role)**을 학생/선생님으로 지정, 학생이면 **담당 선생님**도 지정 → 저장

### ♻️ 달란트 데이터만 초기화 (계정은 유지)
행사를 새로 시작할 때, **계정은 그대로 두고 달란트·기부 기록만 0으로** 비우고 싶다면:
- Render 대시보드 → `holytree-api` → **Shell** 탭에서:
  ```bash
  python manage.py reset_talents --yes
  ```
- 모든 학생의 받은/기부 달란트와 공동체 나무가 0으로 돌아갑니다. (계정·비밀번호는 그대로)

### 🖥️ 행사 때 큰 화면 띄우기
- 행사장 PC/노트북에서 https://holytree.vercel.app/?display 를 열고 **전체화면(F11)** 하면 끝.
- 학생들이 기부할 때마다 숫자와 나무가 자동으로 갱신돼요.

## 8. 운영 팁 & 주의사항

- **⏱️ 첫 접속이 느려요 (정상)**: 백엔드(Render 무료)는 한동안 안 쓰면 잠들어요. 일요일 행사 **몇 분 전에 사이트를 한 번 열어** 깨워두면 첫 사용자가 빠릅니다. (한 번 깨면 그 뒤론 빨라요)
- **📅 8월 결산까지 데이터 보존**: 데이터베이스를 **Neon**에 둔 이유가 이거예요. (다른 무료 서비스는 30일 뒤 데이터가 삭제됨) Neon은 만료가 없어 두 달 내내 안전합니다.
- **🔒 비밀번호 재설정 권장**: 설정 과정에서 Neon 데이터베이스 비밀번호가 한 번 노출됐어요. Neon 대시보드에서 비밀번호를 **재설정(Reset password)** 한 뒤, Render의 `DATABASE_URL` 환경변수를 새 값으로 바꿔주면 더 안전합니다.
- **🌳 나무 영상 교체**: 공동체 나무는 `src/assets/tree-characters/tree_1~4.mp4` 영상으로 자랍니다. 더 멋진 영상이 생기면 같은 이름으로 교체하면 돼요.
